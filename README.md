# Music Recommender

Небольшое приложение-рекомендатель на датасете треков Spotify. Бэкенд - FastAPI-сервис, который на старте грузит ~4500 треков из CSV и отдаёт эндпоинты для популярных треков, топа по любимым исполнителям и поиска. Фронтенд - Streamlit-приложение (UI на русском), которое ходит в бэкенд по HTTP.

Это пока учебная песочница. Никакой ML-модели нет, только сортировка и фильтрация по реальным счётчикам прослушиваний Spotify.

## Что внутри

- `backend/`: FastAPI-сервис. Исходники в `backend/src/`. На старте грузит `songs_spotify.csv`; отдаёт `/api/popular`, `/api/top`, `/api/search`, `/api/artists`.
- `frontend/`: Streamlit-приложение. Исходники в `frontend/src/`. Три страницы: «Топ для тебя» (фильтр по любимым исполнителям), «Популярное сейчас» и «Моя медиатека» (поиск + лайки).
- `docker-compose.yml`: поднимает оба сервиса в одной bridge-сети, чтобы фронт мог достучаться до бэка по имени хоста.

У каждого сервиса свои `requirements.txt`, `.env` и `Dockerfile`, так что ставятся и деплоятся независимо.

## Как запустить

Самый быстрый путь — Docker Compose:

```bash
docker compose up --build
```

Фронт открывается на http://localhost:8501, бэк — на http://localhost:8000 (Swagger по адресу `/docs`).

## Настройки

Оба сервиса читают настройки из `.env`-файла рядом с `Dockerfile`.

- `backend/.env`: `SONGS_CSV_PATH` (необязательное переопределение; по умолчанию используется CSV из репозитория).
- `frontend/.env`: `BACKEND_URL`, `REQUEST_TIMEOUT`.

Переменные окружения процесса перебивают `.env`, так что любое значение можно поменять прямо в шелле, не трогая файлы.

## Данные

`backend/src/data/songs_spotify.csv` - датасет стримов Spotify: ~4600 строк с названием трека, исполнителем, альбомом, числом прослушиваний, флагом explicit и другими полями. Около 4500 остаются после того, как мы выкидываем строки с пропусками в нужных колонках. Сейчас бэкенд читает только несколько из них; с остальными колонками ещё есть с чем поработать.
